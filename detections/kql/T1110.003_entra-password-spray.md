# Entra ID Password Spray

- **Query file:** `T1110.003_entra-password-spray.kql`
- **Author:** nextlayersec
- **Last updated:** 2026-05-27
- **Status:** testing
- **Severity:** High

## What it detects

A single source IP that produces a high number of failed sign-ins against many distinct user principals in a short window — the signature of a password-spray. Successful sign-ins are excluded so the rule fires on the spray itself, not on follow-on activity.

## ATT&CK mapping

- **Tactic:** Credential Access (TA0006)
- **Technique:** T1110.003 – Brute Force: Password Spraying
- **Related:** T1078.004 – Valid Accounts: Cloud Accounts (post-success follow-up)

## Data sources

- Table: `SigninLogs` (Entra ID via diagnostic settings → Log Analytics / Sentinel)
- Useful joins: `IdentityInfo`, `AADRiskyUsers`, `ThreatIntelligenceIndicator`
- Retention recommendation: 90 days

## Logic

Filters Entra sign-ins for known credential-failure `ResultType` codes (50053 account locked, 50126 bad password, 50074 strong-auth required, etc.), then groups by `IPAddress` + `AppDisplayName`. Fires when distinct targeted users ≥ 10 **and** failed attempts ≥ 25 in the lookback window.

## False positives

- Misconfigured on-prem sync agents bombing Entra after a password change.
- Load-testing or vendor IAM tooling.
- NAT egress from a large office (many users behind one IP) hitting MFA simultaneously after a policy change.
- Suppress by allowlisting IP CIDRs / known service principal sources, or by raising `distinctUserThreshold`.

## Tuning

- Adjust thresholds based on tenant size — small tenants should lower `distinctUserThreshold` to 5; large tenants may need 20.
- Add a join on `IdentityInfo` to enrich with department / privilege tier and prioritize alerts targeting admin roles.
- Pivot on `ASN` instead of raw IP for distributed (slow-and-low) sprays from cloud providers.

## Response

See [`blue-team-playbooks/`](../../blue-team-playbooks/) — credential theft / password spray playbook (planned). Containment: block source IP / ASN, force password reset on any user with a corresponding success, expand MFA enforcement.
