# Suspicious LSASS Process Access

- **Query file:** `T1003.001_lsass-access-suspicious.kql`
- **Author:** nextlayersec
- **Last updated:** 2026-05-27
- **Status:** testing
- **Severity:** High

## What it detects

Non-Microsoft, non-EDR processes that open a handle to `lsass.exe` with access rights that enable memory reading or full process control — the precondition for credential dumping (Mimikatz, ProcDump abuse, `comsvcs.dll MiniDump`, direct syscall variants).

## ATT&CK mapping

- **Tactic:** Credential Access (TA0006)
- **Technique:** T1003.001 – OS Credential Dumping: LSASS Memory
- **Related:** T1055 – Process Injection (often paired)

## Data sources

- Table: `DeviceEvents` with `ActionType == "OpenProcessApiCall"` (Defender for Endpoint)
- Supplement: enable Sysmon Event ID 10 if MDE is not present
- Retention recommendation: 30 days

## Logic

Filters `DeviceEvents.OpenProcessApiCall` where the target is `lsass.exe` and the requested rights bit-overlap with `PROCESS_VM_READ` (0x10) or `PROCESS_QUERY_INFORMATION` (0x400). Excludes Microsoft-signed initiating processes (Defender, SenseIR, services) and a hard-coded allowlist of known good binaries.

## False positives

- Endpoint protection products (CrowdStrike, SentinelOne, Carbon Black) — add their signer / image name to `knownGoodImages`.
- Backup and DLP agents that snapshot process memory.
- Microsoft Process Explorer when run by admins for support.

## Tuning

- Replace the image allowlist with a **signer allowlist** via `InitiatingProcessSignatureStatus == "Valid"` and `InitiatingProcessSignerType == "MicrosoftWindowsPublisher"` for fewer maintenance breakages.
- Pivot on `InitiatingProcessFolderPath` — anything launching from `\Users\*` or `\Windows\Temp\` is high-fidelity.
- Join with `DeviceFileEvents` to catch the resulting `.dmp` written by the same `InitiatingProcessId`.

## Response

Containment: isolate host (EDR), force password reset for every user logged in within the previous 24h, rotate machine account / Kerberos TGT. Post-incident: hunt for resulting Pass-the-Hash / Kerberos abuse with sibling detections.
