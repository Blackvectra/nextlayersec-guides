# Run / RunOnce Key Persistence by Non-Installer

- **Query file:** `T1547.001_runkey-persistence.kql`
- **Author:** nextlayersec
- **Last updated:** 2026-06-02
- **Status:** testing
- **Severity:** Medium (raise to High when the writer is in `\Users\*\AppData\*` or has no signer)

## What it detects

A process writing a value to a Windows **Run**, **RunOnce**, **RunOnceEx**, or Winlogon `Shell`/`Userinit` key when the initiating process is NOT a known installer and NOT signed by Microsoft. This is the most common user-mode persistence trick ‚Äî commodity loaders, info-stealers, RAT droppers, and many post-exploitation toolkits all use it.

## ATT&CK mapping

- **Tactic:** Persistence (TA0003)
- **Technique:** T1547.001 ‚Äì Boot or Logon Autostart Execution: Registry Run Keys / Startup Folder
- **Related:** T1112 ‚Äì Modify Registry; T1059.001 ‚Äì PowerShell (frequent writer)

## Data sources

- Table: `DeviceRegistryEvents` (Defender XDR / MDE)
- Equivalent: Sysmon Event ID 13 (`RegistryValueSet`) when MDE is not present
- Retention recommendation: 30 days

## Logic

Filters registry write events whose `RegistryKey` ends with one of the known Run-key suffixes (HKCU + HKLM + Wow6432Node variants + Winlogon `Shell`/`Userinit`). Requires `RegistryValueData` to be non-empty (avoids cleanup deletes). Excludes a small allowlist of known installer / updater binaries (`msiexec.exe`, `setup.exe`, `Update.exe`, GoogleUpdate, EdgeUpdate, etc.) and any initiating process whose signer company name contains "Microsoft".

Outputs the registry path, value name + data, the initiating process tree, and a HKCU-vs-HKLM flag for fast triage.

## False positives

- In-house deployment / packaging tools (Chocolatey, Intune scripts, custom MSI wrappers).
- Vendor agents that pin themselves into Run for legacy reasons (some printer / VPN clients).
- Game launchers and creative-tool agents.
- Suppress by adding the binary to `knownInstallers`, or by joining with `DeviceFileCertificateInfo` and excluding by `Signer` rather than image name.

## Tuning

- **Switch to signer allowlist.** Replace the `knownInstallers` image list with a join on `DeviceFileCertificateInfo` filtering by `IsTrusted == true` and signer organization ‚Äî fewer false negatives, less maintenance.
- **Path heuristic.** Add a high-fidelity branch for writers running from `\Users\*\AppData\Local\Temp\`, `\Users\Public\`, `\ProgramData\`, or `\Windows\Temp\` ‚Äî those are near-certain malware.
- **Value data heuristic.** Alert harder when `RegistryValueData` references `powershell.exe`, `mshta.exe`, `rundll32.exe`, `wscript.exe`, or contains an encoded blob (`FromBase64String`, `-enc`).
- **HKCU vs HKLM.** HKLM writes are higher severity (require local admin); split the alert into two analytics if your triage benefits.

## Response

- Pivot on the initiating process tree to find the originating execution (Office macro, browser download, phish payload).
- Snapshot the registry value before deletion (evidence).
- Hunt the same hash across the fleet with `DeviceProcessEvents` + `DeviceFileEvents`.
