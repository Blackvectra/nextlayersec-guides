# PowerShell Encoded Command Execution

- **Query file:** `T1059.001_powershell-encoded-command.kql`
- **Author:** nextlayersec
- **Last updated:** 2026-05-27
- **Status:** testing
- **Severity:** Medium

## What it detects
PowerShell launches that pass base64-encoded payloads via `-EncodedCommand` (or its `-enc` / `-ec` aliases) or that decode an inline blob with `FromBase64String`. This is a frequent obfuscation pattern for loaders, droppers, and post-exploitation frameworks.

## ATT&CK mapping
- **Tactic:** Execution (TA0002)
- **Technique:** T1059.001 – PowerShell
- **Related:** T1027 – Obfuscated Files or Information

## Data sources
- Table: `DeviceProcessEvents` (Defender XDR / MDE)
- Equivalent for Sentinel: `SecurityEvent` EventID 4688 with `CommandLine`, or `DeviceProcessEvents` via M365D connector.
- Minimum retention recommended: 30 days

## Logic
Filters `powershell.exe` / `pwsh.exe` process starts whose command line contains an encoded-command flag or an inline base64 decode. Extracts the encoded blob into its own column for fast triage. Excludes hosts in the `knownAdminHosts` allowlist.

## False positives
- Configuration tools (SCCM, Intune, Chocolatey, vendor agents) that legitimately run encoded payloads.
- Internal automation pipelines on build/jump hosts.
- Suppress by appending hostnames to `knownAdminHosts` or by joining with `IdentityInfo` to exclude service accounts.

## Tuning
- Tighten the regex `{20,}` length threshold if your environment generates short legitimate blobs.
- Add a join on `DeviceNetworkEvents` to escalate hits that immediately initiate outbound connections.
- Pivot on `InitiatingProcessFileName` — Office apps, `wmiprvse.exe`, `mshta.exe`, or `rundll32.exe` as parents are high-fidelity signals.
