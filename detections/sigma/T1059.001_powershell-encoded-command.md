# PowerShell Encoded Command Execution (Sigma)

- **Query file:** `T1059.001_powershell-encoded-command.yml`
- **Author:** nextlayersec
- **Last updated:** 2026-05-28
- **Status:** experimental
- **Severity:** Medium

## What it detects

PowerShell launches that pass base64-encoded payloads via `-EncodedCommand` (or its `-enc` / `-ec` aliases) or that decode an inline blob with `FromBase64String`. This is a frequent obfuscation pattern for loaders, droppers, and post-exploitation frameworks.

## ATT&CK mapping

- **Tactic:** Execution (TA0002)
- **Technique:** T1059.001 – PowerShell
- **Related:** T1027 – Obfuscated Files or Information

## Data sources

- Log source: `process_creation` (Windows)
- Defender XDR / MDE: `DeviceProcessEvents`
- Sentinel: `SecurityEvent` EventID 4688 with `CommandLine`, or `DeviceProcessEvents` via the M365D connector
- Sysmon: Event ID 1 (process creation)
- Minimum retention recommended: 30 days

## Logic

Matches `powershell.exe` / `pwsh.exe` process starts (`Image|endswith`) whose `CommandLine` contains an encoded-command flag (`-enc`, `-EncodedCommand`, `-ec`) or an inline base64 decode (`FromBase64String`). All terms are OR'd inside the single `selection` block; the rule fires on any one match.

## False positives

- Configuration tools (SCCM, Intune, Chocolatey, vendor agents) that legitimately run encoded payloads.
- Internal automation pipelines on build / jump hosts.

## Tuning

- Add a backend-side allowlist of known admin hosts or service accounts.
- Escalate hits where the parent process is an Office app, `wmiprvse.exe`, `mshta.exe`, or `rundll32.exe` — these are high-fidelity loader signals.
- Convert to your SIEM with `sigma convert -t <backend> detections/sigma/T1059.001_powershell-encoded-command.yml`.
