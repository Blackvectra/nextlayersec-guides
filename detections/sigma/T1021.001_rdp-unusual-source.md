- **Status:** experimental

## What it detects

Successful interactive RDP logons (Windows Security event ID **4624** with `LogonType == 10`) where the source `IpAddress` is **RFC-routable public space** — i.e., RDP from the internet. The regex excludes `10.x`, `127.x`, `169.254.x`, `172.16-31.x`, `192.168.x`, `100.64-127.x` (CGNAT), IPv6 link-local, and IPv6 ULA.

This is the **high-precision external-source branch** of the paired KQL detection. The KQL companion (`detections/kql/T1021.001_rdp-unusual-source.md`) adds two additional branches that need per-environment context to express portably:

- `B_UnexpectedInt` — internal source NOT in admin-subnet allowlist. Requires per-environment data.
- `B_SuccessAfterFails` — success preceded by ≥ 5 failures in 10 minutes. Requires a time-window self-join.

Deploy this Sigma rule directly for the noisy-but-mandatory "RDP from the internet" alert. Layer the KQL on top for the loud-but-tuneable internal pivot detection.

## Data sources

- **Sigma logsource:** `product: windows`, `service: security`.
- **Backends:**
  - **Defender XDR / Sentinel** — `sigma convert -t kusto`. Use the KQL companion for the per-pattern boolean tags + per-environment allowlists.
  - **Splunk** — `sigma convert -t splunk`. Run against `index=wineventlog EventCode=4624`.
  - **Elastic** — `sigma convert -t elasticsearch`. Maps to `winlog.event_data.LogonType` / `winlog.event_data.IpAddress`.

## False positives

- Site-to-site partner / vendor maintenance from a fixed public IP — add an `IpAddress` exclusion in your fork, document why.
- Pen-test / Red Team engagements — pause the alert OR temporarily allowlist the engagement source IPs.
- Mis-configured admin sessions without VPN — fix the policy, don't suppress the alert.

## Tuning

- **`IpAddress` exclusion list** — add known-good partner / vendor / SD-WAN concentrator IPs as `IpAddress|notcontains:` items in your fork.
- **Tier dial** — if your fleet has bastion hosts that legitimately accept inbound RDP from the internet (cloud-hosted helpdesk jump boxes), add a `TargetUserName` selector that scopes the alert to non-admin accounts only on those hosts, and fires a high-severity alert if any tier-0 admin account logs in.
